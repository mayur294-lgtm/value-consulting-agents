Orchestrator & Engagement Workflows

The Value Consulting Orchestrator is the central router. It classifies every engagement into one of three types and routes to the appropriate agent pipeline.

Value Consulting Orchestrator

Detects engagement type, gathers inputs, routes to agents, enforces context protocol, manages journal

Value Assessment

Evidence-Based

Post-discovery transcripts and financial data feed a 6-agent pipeline producing ROI models, capability assessments, and roadmaps.

Ignite Inspire

Workshop-Driven

4 workshop prep modes, synthesis, use case design, prototyping, and presentation — ending with an Ignite Day package.

Hybrid

Assessment + Inspire

Combines evidence-based ROI with workshop-driven vision. Cross-references findings for maximum defensibility.

The core 6-agent pipeline. Each agent reads only the output of the previous agent (never raw transcripts), following the disk-based handoff protocol.

Consistency audit + executive-ready deliverables

Discovery Transcript Interpreter Sonnet
Extracts structured evidence from raw transcripts and interview notes. Processes one file at a time with chunking for large inputs.
Input
Raw transcripts, interview notes, unstructured client data
Output
Evidence Register (E1..En), Pain Point Register, Metric Register, Constraints, Open Questions
Capability Assessment Sonnet
Scores current-state maturity (1-5), identifies gaps, maps improvements to value levers. Journey-based structure: Acquire → Activate → Expand → Retain.
Input
Evidence Register, domain context, benchmarks
Output
Maturity scores, gap analysis, value driver mapping, servicing task analysis
ROI Business Case Builder Sonnet
Builds defensible financial models. Every benefit maps to evidence, every assumption is documented. Three scenarios with sensitivity analysis.
Input
Evidence Register, Capability Assessment, financial data, benchmarks
Output
Excel ROI model (8 sheets), Executive Summary Report, Assumptions Register
Roadmap Prioritization Sonnet
Sequences initiatives by dependency logic, value logic, risk logic, and capacity logic. Creates initiative cards with dependencies, decision gates, and value milestones.
Input
Capability gaps, ROI value levers, domain context, constraints
Output
Phased roadmap, initiative cards, dependency map, value realization timeline
Narrative Assembler Sonnet
Final quality gate. Runs a consistency audit across all upstream artifacts (Does ROI timeline match Roadmap? Do Capability priorities align? Are Pain Points mapped to Gaps?). Produces the deliverable package with confidence rating (HIGH/MEDIUM/LOW) and marks the engagement COMPLETE in the journal.
Input
All upstream outputs: Discovery synthesis, Capability Assessment, ROI Model, Roadmap
Output
Executive Summary (1 page max), Assessment Report, consolidated Assumptions Register, ENGAGEMENT_JOURNAL.md marked COMPLETE

Workshop Preparation Sonnet
Prepares hypothesis-driven facilitation materials. 4 modes: Strategy (competitive research), Member/CX (persona research via social media & app reviews), Employee/EX (systems & productivity), Architecture (tech landscape & integration patterns).
Input
Client context, strategy docs, annual reports
Output
Workshop decks (HTML), personas, hypotheses, competitive analysis, ENGAGEMENT_CONTEXT.md
Ignite Workshop Synthesizer Sonnet
Consolidates all 4 workshop outputs. Builds hypothesis validation matrix, cross-workshop patterns, and priority-scored use case candidates using: Value Impact (40%) + Strategic Alignment (30%) + Feasibility (30%).
Input
All 4 workshop outputs + ENGAGEMENT_CONTEXT.md
Output
Synthesis report, Use Case Candidates, ROI Inputs Package, hypothesis validation matrix
Use Case Designer Sonnet
Designs detailed use cases validated against the Backbase Product Directory. Maps every feature to Product IDs (RB.X.X.X), classifies as OOTB / Config / Extension / Custom / Innovation. Links to Value Proposition themes and Customer Lifestages. Validates against the 10-layer Backbase Architecture.
Input
Use Case Candidates from Synthesizer, Product Directory CSV, domain personas, value propositions
Output
10-section Use Case Documents with Product Directory gap analysis (% OOTB vs Config vs Custom), Architecture alignment, screen flows
+

Support Agent

Benchmark Librarian Sonnet
Curates industry & regional benchmarks from the registry and domain libraries. Every benchmark may end up in board documents — accuracy is non-negotiable. Enforces confidence levels: High (same region + domain, n>50, <2yr), Medium (region OR domain proxy, 20% haircut), Low (global proxy, >4yr). Called by ROI and Capability agents.
Input
Engagement context (domain, region, specific metrics needed)
Output
Curated benchmark shortlist with value, source citation, confidence level, match type, caveats, gaps identified

Context Management Protocol

The critical innovation that makes multi-transcript engagements reliable. Prevents context overflow by enforcing sequential processing and disk-based handoffs.

The Rule: Never load more than one transcript at a time

1. Check Size <500 lines: process whole
500-1500: 2-3 chunks
>1500: 400-500 line chunks
2. Process & Write Extract evidence from chunk
Write findings to disk
Release context
3. Next Chunk Load next chunk
Append to disk file
Repeat until complete
4. Downstream Agents Read consolidated outputs only
Never raw transcripts
Progressively distilled data

Engagement Journal

Persistent engagement memory. Every agent appends an entry when it completes. Enables 3-month resume without reprocessing, audit trail for compliance, and institutional memory.

Journal Entry Structure

Date/Time — Agent Name — Action taken — Files read — Files produced — Key decisions — Assumptions made — Status — Open items

Q

Quality Standards

Executive-Ready

C-level audience. Clear, concise, jargon-free. Action-oriented.

Evidence-Based

All claims sourced. Methodology visible. Assumptions documented.

Decision-Oriented

Clear Go/No-Go. Risk-aware. Next steps defined.

Conservative

Underestimate benefits. Overestimate costs. Show downside case.

Non-Negotiable Rules

RuleStandard
Evidence TraceabilityEvery claim maps to Evidence ID(s). No orphan claims.
Assumption HandlingAcknowledge gap → Conservative assumption → Document → Flag for validation → Sensitivity test ±25-50%
Benchmark ConfidenceHigh (region+domain+recent), Medium (proxy, 20% haircut), Low (global proxy, flagged)
Financial PrecisionMillions to nearest $100K. No false precision. Every number has units and currency.
Scenario ModelingThree scenarios required. Conservative < Moderate < Aggressive. Always.
ROI TraceabilityEvery initiative maps to at least ONE capability gap AND ONE value lever. If not → question or remove.
