Your Role (and Claude's Role)

The claudevc project is a consulting toolkit powered by Claude AI. It generates ROI (Return on Investment) models — financial reports that help bank executives decide whether to invest in Backbase software.

Your job is to make these outputs accurate and professional. But here's the important part:

You do NOT need to touch any code or figure out which files to change. Your job is to test the outputs, spot problems, and describe them in plain English. Claude will figure out which files need to change and make all the fixes for you.
Your Job (Mariam)

Test & Review

  • Run test scenarios
  • Open and review the Excel files and reports
  • Spot errors, missing info, or bad numbers
  • Describe the problem in plain English to Claude
  • Verify the fix worked
Claude's Job

Find & Fix

  • Figure out which file(s) need to change
  • Edit the Python code, agent instructions, templates, or benchmarks
  • Handle all technical work
  • Explain what it changed and why
  • Save changes to git when you ask

Think of yourself as a quality reviewer and Claude as the developer who fixes what you find. You work together as a team.

Step 2: Review the Output

Open the generated files and check them using the lists below. You are looking for anything that looks wrong, missing, or unrealistic.

Check the Excel File (.xlsx)

Open it in Excel or Google Sheets, then look for:

  • Cover Page — Does it show the correct client name, date, and currency?
  • Results Dashboard — Are the big numbers (ROI, NPV, payback) in a reasonable range?
    Expected: ROI 60-200%, Payback 1.5-3.5 years depending on banking type
  • Cashflows sheet — Do the yearly numbers add up to the totals? Try adding a few rows yourself with a calculator.
  • Assumptions sheet — Does EVERY number have a source? Look for any blank "Source" cells.
  • Servicing Analysis — Do the staff volumes, time-per-task, and costs look like real-world numbers?
  • Scenarios — Are there three scenarios (Conservative, Moderate, Aggressive)?
  • Implementation curves — Does Year 1 show the lowest percentage and Year 5 the highest?
  • Formatting — Are input cells highlighted in blue? Does the file open without errors?

Check the ROI Report (.md)

Open the markdown report in VS Code (just click on it in the sidebar). Look for:

  • Executive Summary — Is there a clear Go / No-Go / Conditional Go recommendation?
  • Benefits — Is each benefit linked to a source (evidence or benchmark)?
  • Assumptions — Are they listed with confidence levels (High / Medium / Low)?
  • Sensitivity analysis — Does it show what happens if key numbers are wrong?
  • "What Would Change the Decision" — Is this section included?
  • Currency labels — Are all dollar/euro amounts labeled (USD, EUR, etc.)?
  • Quality — Does it read like a senior consultant wrote it? Would you feel confident showing it to a bank executive?

Check Missing Data Handling (Scenario 3 especially)

  • Are data gaps called out clearly? (e.g., "Customer count not provided")
  • Are conservative assumptions made and documented? (not optimistic guesses)
  • Is each assumption flagged for validation? (e.g., "To be confirmed with client CFO")
  • Does the sensitivity analysis show what happens if assumptions are off by 25-50%?

Step 3: Tell Claude to Fix It

When you find a problem, describe it in plain English in the Claude chat. You do NOT need to know which file to change — Claude will figure that out and make the fix.

The Golden Rule Just describe WHAT is wrong. Don't worry about WHERE or HOW to fix it. Claude handles all of that.

How to Describe Problems Well

Be specific. Say what you expected to see vs. what you actually saw. Here are real examples:

WRONG NUMBERS
You opened the Excel and the ROI shows 450% which is unrealistically high.
The ROI model I just generated shows an ROI of 450%. That's way too high —
for a retail bank it should be in the 60-150% range. The benefits seem
inflated. Please investigate and fix this so the numbers are more realistic
and conservative.copy & paste & adapt
MISSING INFORMATION
The Assumptions sheet has rows where the "Source" column is blank.
I'm looking at the Assumptions sheet in the Excel file and rows 8, 12, and
15 have no source listed. Every assumption needs a source — either a
benchmark reference, "client-provided", or "estimate - to be validated".
Please fix this so no assumption is ever left without a source.copy & paste & adapt
DOUBLE COUNTING
You notice the same saving appears in two different levers.
I think there's double counting in the ROI model. The "RM time saved" benefit
appears in both Lever 2 (Onboarding) and Lever 4 (Servicing). The same
30% time reduction is being counted twice. Please fix this so each benefit
is only counted once.copy & paste & adapt
FORMULAS NOT ADDING UP
The yearly numbers on the Cashflows sheet don't match the total.
On the Cashflows sheet, the Year 1 through Year 5 benefit values are
$200K, $500K, $800K, $1M, and $1.2M. That should total $3.7M but the
"Total Benefits" cell shows $4.1M. The formula seems wrong. Please fix it.copy & paste & adapt
REPORT QUALITY
The executive summary reads like a technical document instead of a business document.
The executive summary in the ROI report uses too much technical jargon like
"STP rates" and "API integrations". A bank CEO would not understand this.
Please rewrite the instructions so the report always uses plain business
language that a non-technical executive can understand.copy & paste & adapt
MISSING SECTION
The report doesn't include the "What Would Change the Decision" section.
The ROI report is missing the "What Would Change the Decision" section.
This section should list the conditions under which the investment would
become a No-Go. It's required in every report. Please fix the system so
this section is always included.copy & paste & adapt
WRONG BENCHMARKS
The model used global benchmarks when regional ones were available.
The model for the Southeast Asia retail bank used global benchmarks for
digital adoption rates. We have Southeast Asia-specific benchmarks that
should be used instead. When regional benchmarks exist, the system should
always prefer those over global ones. Please fix this.copy & paste & adapt
SCENARIOS DON'T MAKE SENSE
The Conservative scenario shows higher ROI than Moderate in some areas.
In the Scenarios sheet, the Conservative scenario shows a Year 3 benefit
of $1.5M but the Moderate scenario shows only $1.2M for the same year.
Conservative should always be lower than Moderate, which should always be
lower than Aggressive. Please fix this.copy & paste & adapt
EXCEL FILE ERRORS
The Excel file shows an error or broken formula when you open it.

#REF! errors in cells C5 and C8. The formulas seem to be referencing
cells that don't exist. Please fix the Excel generator so these
formulas work correctly.copy & paste & adapt

Launch Readiness Checklist

The ROI system is ready for launch when ALL three test scenarios consistently pass ALL of these checks. Run through each scenario at least twice to make sure results are consistent.

Numbers & Calculations

  • ROI percentages are in the expected range for the banking segment
  • NPV, IRR, and payback period look correct
  • Year-by-year cashflows add up to the totals shown
  • Implementation curves go from low (Year 1) to high (Year 5)
  • Conservative < Moderate < Aggressive — always, for every metric
  • No benefit is counted twice across different levers
  • All money amounts have a currency label (USD, EUR, etc.)

Assumptions & Transparency

  • Every number has a source (transcript, benchmark, or labeled assumption)
  • Every assumption has a confidence level (High / Medium / Low)
  • Every assumption has a validation owner ("To be confirmed with CFO")
  • When data is missing, the system uses conservative (safe/low) estimates
  • Sensitivity analysis covers the top 3-5 most important variables

Benchmarks

  • Regional benchmarks used when available (not global ones)
  • When only global benchmarks are available, a 20% conservative haircut is applied
  • Benchmark sources and dates are cited
  • Benchmarks are recent (less than 4 years old)

Report Quality

  • Executive summary lets someone make a decision without reading the full report
  • Clear Go / No-Go / Conditional Go recommendation
  • Benefits-to-evidence mapping table is complete
  • "What Would Change the Decision" section is present
  • Language is plain business English (no tech jargon)

Excel Quality

  • All expected sheets are present (Cover, Instructions, Dashboard, Inputs, Cashflows, Servicing, Scenarios, Assumptions)
  • Input cells are clearly marked (blue background)
  • No broken formulas or #REF! errors
  • Charts display correctly
  • File opens cleanly in both Excel and Google Sheets

If You Get Stuck

Claude knows the entire project. If you're unsure about anything, just ask. Here are some things you can say:

UNDERSTANDING A NUMBER
You see a number in the Excel but don't know if it's reasonable.
Is an NPV of $15 million realistic for a wealth management firm
with 50 relationship managers? That seems high to me.
NOT SURE WHAT TO CHECK
You've reviewed the obvious things but want to dig deeper.
I've checked the main numbers and they look OK, but what else should
I look for? Are there common accuracy problems I might be missing?
SOMETHING LOOKS OFF BUT NOT SURE WHY
You have a gut feeling something is wrong but can't pinpoint it.
The servicing analysis numbers don't feel right. The cost avoided
per year seems too high for a bank this size. Can you double-check
the assumptions and calculations in the servicing section?
COMPARING TO REAL EXAMPLES
You want to validate against actual past models.
Can you compare the ROI model we just generated against the real
Seabank and HNB examples in the project? Are our numbers in line
with those actual engagements?
GIT PROBLEMS
Something isn't working with saving or pushing.
I'm getting an error when trying to push to GitHub. Can you help
me figure out what's wrong and fix it?
STARTING A NEW SESSION
You're coming back to work after a break or a new day.
I'm continuing my ROI improvement work. Please pull the latest
changes from GitHub, check what branch I'm on, and show me what
we've changed so far. Then I'll run the next test.

Glossary

Terms you'll see in the ROI outputs and this guide:

ROI
Return on Investment — how much profit you get back compared to what you spent
NPV
Net Present Value — the total value of future money in today's terms ($1 today is worth more than $1 next year)
IRR
Internal Rate of Return — the annual growth rate an investment is expected to generate
Payback Period
How many years until the investment pays for itself
Lever
A specific way value is created (e.g., "faster onboarding" or "reduced servicing cost")
Benchmark
An industry average used as a reference point for calculations
Assumption
A value we use when real data is not available, labeled as an estimate
Sensitivity Analysis
Testing what happens to the result if key assumptions turn out to be wrong
Conservative
Using lower/safer estimates rather than optimistic ones
Scenario
A set of assumptions — Conservative (cautious), Moderate (likely), Aggressive (optimistic)
FTE
Full-Time Equivalent — a way to measure staffing (1 FTE = 1 full-time person)
AUM
Assets Under Management — total value of investments a firm manages
HNW
High Net Worth — wealthy individual clients
RM
Relationship Manager — the banker who manages client relationships
Git
Version control — tracks changes to files so nothing is lost. Claude handles this for you.
Pull Request
A request to merge your improvements into the main project, sent to Mayur for approval
Branch
Your own separate workspace. Changes stay here until approved. Claude manages this for you.
Commit
A saved snapshot of your changes. Like pressing "Save" but with a description. Claude does this for you.
