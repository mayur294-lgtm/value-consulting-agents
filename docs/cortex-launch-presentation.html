Getting Started

Starting Your First Engagement

$ ./scripts/init_engagement.sh acme_bank 2026-02_retail_assessment

This creates everything you need:

engagements/acme_bank/
  2026-02_retail_assessment/
    inputs/ ← Drop your transcripts here
    outputs/ ← Magic appears here
    ENGAGEMENT_JOURNAL.md ← Your engagement memory
    engagement_intake.md ← Fill in client context
  CLIENT_PROFILE.md ← Persists across engagements!

That's it. Drop transcripts in inputs/ and tell Cortex to go.

Insider Knowledge

Pro Tips from Day 1

1
Always start from cortex/
This is critical. Agents only activate when you're in the Cortex directory. Weird behavior? Check this first.
2
Let Cortex ask YOU questions
Don't try to front-load everything. Start simple, and Cortex will ask for what it needs. It's a conversation.
3
YOU own every output
Cortex drafts, you validate and own. If you can't explain it to the client, it doesn't ship. Review every deliverable — that's why you're the consultant.
4
CLIENT_PROFILE.md is your memory
It persists across engagements. Second engagement with the same client? Cortex already knows them.
5
/domain-context is your secret weapon
Run this before ANY engagement. It loads benchmarks, pain points, journeys, and value props for the specific banking domain. Your deliverables will be 10x richer.

Don't Panic

When Things Go Wrong

(They won't. But just in case.)

"It's not finding my agents!"
Are you in the cortex/ directory? cd cortex and try again. This is the #1 gotcha.
"The output looks weird"
Ask Cortex to "run the 7-step review checklist". It'll self-correct inconsistencies between deliverables.
"How do I update Cortex?"
git pull — that's literally it. You'll get release notes by email too.
"I want to customize an agent"
Talk to your Cortex Champions first (Mariam, Anton, Shobhit, Mayur). Agents are calibrated to work together.
Need help?
Drop a message in 🔒 backbase-value-consulting on Slack. We've got you. No question is too basic.
