Keeping Your Agents Updated

A 2-minute guide for consultants on pulling the latest agent definitions, knowledge packs, and slash commands.

01 Why Updates Matter

The repo IS the system. When agents improve, you need those improvements locally.

⚡ What lives in the repo

  • 18 specialized agents — discovery, capability assessment, ROI, roadmap, narrative assembler, and more
  • 11 slash commands — /presentation, /generate-roi-excel, /prototype, etc.
  • Domain knowledge packs — retail, SME, commercial, wealth, corporate benchmarks and personas
  • Templates & methodology — engagement structures, journal format, intake forms
  • CLAUDE.md — the system prompt that makes Claude behave as a Value Consultant
!
If your local repo is behind, your agents are stale. You may be running analysis with outdated prompts, missing new capabilities, or producing deliverables that don't match the latest quality standards.

02 The Quick Update (90 seconds)

Run these commands before starting any engagement.

Open your terminal and navigate to the Cortex directory

This is where you cloned the repo. If you followed setup, it's in your Documents folder.

Terminal
cd ~/Documents/cortex

Check your current status

Make sure you don't have uncommitted local changes that could conflict.

Terminal
git status
i
You should see "nothing to commit, working tree clean". If you see modified files, see the FAQ section below.

Pull the latest changes

This fetches and merges all updates from the main branch in one command.

Terminal
git pull origin main

Verify the update

Quick sanity check to confirm you're on the latest version.

Terminal
cat VERSION
# Should show the latest version number

git log --oneline -5
# Shows the 5 most recent commits
✓
That's it. Open Claude Code (VS Code extension or CLI) from the cortex/ directory and all updated agents, commands, and knowledge are loaded automatically. No restart or configuration needed.

03 Copy-Paste One-Liner

For the busy consultant who just wants to update and go.

Terminal — all-in-one
cd ~/Documents/cortex && git pull origin main && echo "Updated to $(cat VERSION) — $(git log --oneline -1)"

This navigates to the repo, pulls the latest, and prints a confirmation with the version number and latest commit message.

After pulling, see what's new before jumping into an engagement.

08 Troubleshooting & FAQ

Common questions and how to resolve them.

"git pull" says I have local changes that would be overwritten ▼

This means you've accidentally edited files in the repo (agent definitions, templates, etc.). To fix:

Terminal
# First, see what you changed
git diff

# If you don't need your changes, reset and pull
git stash && git pull origin main

# If you want to keep your changes for later
git stash
git pull origin main
git stash pop
# Then resolve any conflicts
"git pull" shows a merge conflict ▼

This is rare but can happen. The safest approach for consultants (who shouldn't be modifying agent definitions locally):

Terminal — Nuclear option (safe)
# Abort the merge
git merge --abort

# Reset to match remote exactly
git fetch origin
git reset --hard origin/main
!
This discards any local modifications to repo files. Your engagements/ folder is NOT affected (it's gitignored).
I'm getting "permission denied" or authentication errors ▼

You need GitHub access to the repository. Contact your team lead to ensure:

  • You've been added as a collaborator on the repo
  • You've authenticated Git with GitHub (via SSH key or personal access token)
  • Run gh auth status to check if you're logged in with the GitHub CLI
My slash commands aren't showing up after pulling ▼

Make sure you're opening Claude Code from the cortex directory. The system only activates when .claude/ and CLAUDE.md are in your working directory.

Terminal
# Make sure you're in the right directory
pwd
# Should show: .../cortex

# Check the files exist
ls .claude/agents/ | wc -l
# Should show: 18
Will git pull delete my engagement data? ▼

No. The engagements/ folder is in .gitignore. Git does not touch untracked or gitignored files. Your client transcripts, outputs, and journals are completely safe.

Do I need to install new Python dependencies after pulling? ▼

Only if you use the Excel ROI export feature. If in doubt, run it after pulling — it's quick and harmless:

Terminal
pip install -r requirements.txt
