UC-M01

Effortless Account Opening

Priority: P1 Category: Tablestakes Module: Digital Onboarding Stage: Welcome & Access

2. Business Context

Strategic Alignment: Directly supports "Digital-First Islamic Banking" — the bank's flagship digital transformation theme.

Business Value: Reduce cost per acquisition by 60%; increase digital acquisition from 15% to 50% of new accounts; reduce abandonment rate from estimated 70% to under 30%.

Current State: Customers must visit a branch to complete account opening. A partial digital form exists but requires a branch visit for ID verification and Shariah consent signing.

Pain Points:

  • "I started online but had to visit a branch to finish" — top CX workshop complaint
  • Average 3-day completion time for what should be a 10-minute process
  • High drop-off for younger, digitally-native customers

3. User Context

Primary Persona: Young Professional Khalid (28, tech-savvy, first Islamic banking relationship)

User Goal: Open a Shariah-compliant current account entirely from my phone in under 10 minutes.

Success Criteria: Account is open, IBAN is received, debit card is ordered, mobile banking is immediately accessible.

4. Happy Path Specification

Preconditions: Customer has Emirates ID, smartphone with camera, and is a UAE resident.

Trigger: Customer taps "Open Account" on DIB website or app store listing.

StepUser ActionSystem Response
1Taps "Open Account"Displays welcome screen with Islamic banking proposition and requirements
2Selects product typeShows Shariah-compliant account options with features and profit rates
3Enters personal informationValidates format, checks eligibility in real-time
4Scans Emirates IDOCR extracts data, auto-fills form, verifies with ICA
5Reviews details and accepts Shariah disclosureDisplays Mudarabah/Wakalah terms and consent checkbox
6Confirms and submitsCreates account in T24, generates IBAN, orders card
7Views confirmationShows IBAN, card ETA, and immediate mobile banking access

Postconditions: Account created in T24, IBAN generated, debit card ordered, welcome SMS/email sent, customer can immediately access DIB mobile banking.

5. Screen Flow

  • Screen 1: Welcome — Islamic banking value proposition and requirements
  • Screen 2: Product Selection — Choose account type (Current, Savings, Al Islami)
  • Screen 3: Personal Information — Name, contact, nationality
  • Screen 4: Emirates ID Capture — Camera scan and OCR
  • Screen 5: Review & Shariah Consent — Summary and Islamic disclosure
  • Screen 6: Processing — Real-time account creation status
  • Screen 7: Success — IBAN, card order confirmation, next steps

6. Data Requirements

Input: Full name, Emirates ID number, nationality, date of birth, mobile number, email, Emirates ID images.

System: ICA verification result, AML/sanctions screening, product catalogue, profit rate schedule.

Output: Account number, IBAN, customer record in T24, card order reference.

7. Integration Requirements

SystemIntegrationPurpose
Temenos T24Real-time API via HubAccount creation and customer record
ICA (Emirates ID)Real-time APIIdentity verification
AML/SanctionsReal-time APICompliance screening
Card ManagementReal-time API via HubDebit card issuance
Notification ServiceEvent-drivenWelcome SMS/email

8. Backbase Implementation

Module: Digital Onboarding

Approach: Configuration + Extension (Shariah consent step is extension)

Key Components: Onboarding Journey Orchestrator, Identity Verification Widget, Document Capture Widget, E-Signature Widget

Customization: Shariah disclosure and consent UI, Emirates ID OCR integration, DIB branding, T24-specific integration layer, Arabic language support

9. Definition of Done

  • Customer can complete account opening 100% digitally in under 8 minutes
  • Emirates ID verification completes within 30 seconds
  • Shariah consent is digitally captured and stored for audit
  • Account is created in T24 within 60 seconds of submission
  • Customer can access mobile banking immediately after account creation
  • Abandonment rate below 30%

UC-M02

Instant Shariah-Compliant Financing

Priority: P1 Category: Differentiating Module: Digital Lending Stage: Grow & Finance

2. Business Context

Strategic Alignment: Supports "Digital-First Islamic Banking" and "Customer Lifecycle Value" by enabling instant financing decisions.

Business Value: Reduce financing application time from 5 days to under 15 minutes; increase digital origination from 8% to 40%; reduce cost per origination by 45%.

Current State: Customers fill a paper form in-branch or a basic web form. Average decision time is 3-5 working days with multiple document submissions.

Pain Points:

  • "I applied for financing a week ago and still haven't heard back"
  • Customers must provide the same documents multiple times
  • No real-time calculator showing Shariah-compliant payment structure

3. User Context

Primary Persona: Family Provider Omar (38, mid-career, needs personal financing for home renovation)

User Goal: Get an instant decision on Shariah-compliant personal financing with clear Murabaha terms, entirely from my phone.

Success Criteria: Financing offer received with clear profit rate and repayment schedule; funds disbursed within 24 hours of acceptance.

4. Happy Path Specification

Preconditions: Customer is an existing DIB account holder with active mobile banking.

Trigger: Customer taps "Apply for Financing" in DIB mobile app.

StepUser ActionSystem Response
1Taps "Apply for Financing"Shows welcome with pre-approved amount (if eligible)
2Selects financing typeDisplays Murabaha / Ijarah product options
3Uses financing calculatorReal-time calculation with Shariah-compliant profit rate
4Enters employment and salary detailsPre-fills from existing data, validates
5Uploads salary certificateOCR extracts data, validates against entered info
6Reviews Shariah disclosure and termsShows Murabaha contract terms and total cost
7Submits applicationRuns credit scoring and affordability check
8Views decisionShows approved amount, profit rate, repayment schedule

Postconditions: Application created in LOS, credit decision recorded, Shariah terms accepted, funds disbursed to account within 24 hours.

5. Screen Flow

  • Screen 1: Welcome — Pre-approved amount and financing options
  • Screen 2: Product Selection — Murabaha / Ijarah options
  • Screen 3: Financing Calculator — Amount, tenure, monthly payment
  • Screen 4: Employment Details — Salary, employer, tenure
  • Screen 5: Document Upload — Salary certificate
  • Screen 6: Shariah Disclosure — Murabaha terms and consent
  • Screen 7: Processing — Credit decision
  • Screen 8: Approval — Congratulations with offer details

6. Data Requirements

Input: Financing amount, tenure, employment details, salary certificate, consent.

System: Pre-approved limit, credit score, affordability model, Shariah profit rates, existing liabilities.

Output: Financing application, credit decision, Murabaha contract, repayment schedule.

7. Integration Requirements

SystemIntegrationPurpose
Temenos T24Real-time API via HubCustomer data, account balances, liabilities
Loan Origination SystemReal-time APIApplication creation and decisioning
Credit Bureau (Al Etihad)Real-time APICredit score and report
Document ManagementReal-time APISalary certificate storage

8. Backbase Implementation

Module: Digital Lending

Approach: Configuration + Extension (Shariah calculator and disclosure are extensions)

Key Components: Lending Journey Orchestrator, Affordability Widget, Document Upload Widget, Decision Widget

Customization: Murabaha profit rate calculator, Shariah disclosure UI, Al Etihad credit bureau integration, Arabic support

9. Definition of Done

  • Customer receives financing decision within 5 minutes of submission
  • Shariah-compliant profit structure clearly displayed before consent
  • Salary certificate OCR achieves 90%+ accuracy
  • Funds disbursed within 24 hours of acceptance
  • Complete audit trail of Shariah consent captured

UC-M03

Family Financial Hub

Priority: P1 Category: Differentiating Module: Digital Banking Stage: Manage & Protect

2. Business Context

Strategic Alignment: Supports "Customer Lifecycle Value" — deepens household relationships and increases products per family.

Business Value: Increase products per household from 2.1 to 3.5; reduce churn of secondary family members; enable AED 500M+ in family savings goals.

Current State: Family members have individual accounts with no shared view. Parents cannot monitor children's spending. No shared financial goals.

Pain Points:

  • "I want to teach my children about money but can't see their spending"
  • "We save for Hajj as a family but track it in a spreadsheet"
  • No way to set spending limits on children's supplementary cards

3. User Context

Primary Persona: Family Provider Omar (38, married with 2 children, wants to manage family finances holistically)

User Goal: See all family accounts in one view, set shared savings goals, and manage children's allowances from my phone.

Success Criteria: Family dashboard shows all linked accounts, shared goals track progress, children receive allowances automatically.

4. Happy Path Specification

Preconditions: Customer is an existing DIB account holder with at least one family member also banking with DIB.

Trigger: Customer taps "Family Hub" in DIB mobile app main menu.

StepUser ActionSystem Response
1Opens Family HubShows family dashboard with linked members
2Adds a family memberSends invitation via SMS/app notification
3Creates a shared goalSets up goal with target amount and timeline
4Views family budgetShows combined spending by category across family
5Configures child allowanceSets recurring transfer and spending limits
6Reviews notificationsShows spending alerts and goal milestones

Postconditions: Family members linked, shared goals tracking, allowances scheduled, spending notifications active.

5. Screen Flow

  • Screen 1: Family Dashboard — Members, combined balance, alerts
  • Screen 2: Add Family Member — Invite via SMS/notification
  • Screen 3: Shared Goals — Create and track family goals
  • Screen 4: Family Budget — Combined spending categories
  • Screen 5: Child Allowance — Configure recurring and limits
  • Screen 6: Notifications & Settings — Alerts and preferences

6. Data Requirements

Input: Family member invitations (phone/account), goal names/amounts/dates, allowance amounts/frequency, spending limit settings.

System: Account balances for all linked members, transaction data for spending categories, goal progress calculations.

Output: Family group record, shared goal accounts, scheduled transfers, notification preferences.

7. Integration Requirements

SystemIntegrationPurpose
Temenos T24Real-time API via HubAccount balances and transactions
Card ManagementReal-time API via HubSupplementary card limits
Notification ServiceEvent-drivenSpending alerts and goal milestones

8. Backbase Implementation

Module: Digital Banking (with Engage for notifications)

Approach: Extension (family grouping and goal tracking are extensions)

Key Components: Account Aggregation Widget, Goal Tracker Widget, Transfer Scheduler, Notification Engine

Customization: Family member linking flow, shared goal UI, child allowance logic, Islamic values-aligned goal templates (Hajj, Umrah, Zakat)

9. Definition of Done

  • Parent can view all family member accounts in a single dashboard
  • Shared goals track contributions from multiple family members
  • Child allowances are scheduled automatically on set dates
  • Spending limits on supplementary cards enforced in real-time
  • Notifications delivered within 30 seconds of triggering event
