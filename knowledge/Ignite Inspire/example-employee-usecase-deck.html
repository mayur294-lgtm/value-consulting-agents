UC-E01

One View, Every Context

Priority: P1 Category: Tablestakes Module: Digital Assist Capability: 360° View + Omnichannel Context

2. Business Context

Strategic Alignment: Directly enables "Operational Efficiency" by eliminating the 6-system lookup currently required for a single customer inquiry.

Business Value: Reduce average inquiry handling time by 65% (from 4.5 min to 1.5 min), save $1.8M annually in switching tax, increase first-contact resolution from 62% to 85%.

Current State: Relationship managers and contact center agents toggle between T24, CRM, card system, document imaging, digital banking admin, and email to assemble a customer view. No visibility into customer's digital journey.

Pain Points Addressed:

  • "I ask the customer to wait while I check 5 different systems."
  • "A customer called about their online financing application and I couldn't see it."
  • "Every time a customer is transferred, they have to repeat everything."

3. User Context

Primary Persona: Noor Al-Fahim — Relationship Manager, branch-based, serves 200+ customers, needs instant context.

Secondary Personas: Saeed Al-Hashemi (Contact Center), Rania Mahmoud (Financing Specialist)

User Goal: See everything about a customer in one screen — accounts, history, digital activity, open cases, and relevant context — before or during the conversation.

Success Criteria: Complete customer context available within 3 seconds, digital journey visible, interaction notes saved and shared across channels.

4. Happy Path Specification

Preconditions: Employee is authenticated in Digital Assist. Customer record exists.

Trigger: Employee searches for customer or customer is auto-identified (inbound call, appointment).

StepEmployee ActionSystem Response
1Searches customer name or IDDisplay customer 360 dashboard
2Views digital context tabShow customer's recent app/web activity, abandoned journeys
3Reviews interaction historyShow timeline of all interactions across channels
4Initiates quick actionOpen pre-filled workflow (e.g., address change, card block)
5Adds interaction noteSave note linked to customer record, visible to all channels
6Completes or hands offUpdate case status, warm transfer with full context

Postconditions: Interaction recorded, customer context updated, case progressed, notes visible to next agent.

5. Screen Flow

  • Screen 1: Customer Search — Search bar with recent customers
  • Screen 2: 360 Dashboard — Accounts, balances, key data, alerts
  • Screen 3: Digital Context — Recent app activity, abandoned journeys
  • Screen 4: Interaction History — Timeline across all channels
  • Screen 5: Quick Actions — Pre-filled common tasks
  • Screen 6: Notes & Handoff — Save notes, warm transfer

6. Data Requirements

Input Data: Customer search query (name, ID, phone, IBAN).

System Data: Account balances (T24), product holdings, transaction history, card status, digital session data, CRM interaction log, open cases.

Output Data: Interaction notes, case updates, action triggers.

7. Integration Requirements

SystemIntegration TypePurpose
Temenos T24Real-time API (via Hub)Account balances, product holdings, transaction history
Card ManagementReal-time API (via Hub)Card status, spending data
CRMReal-time API (via Hub)Interaction history, customer notes
Digital Banking PlatformEvent-driven (via Hub)Digital session data, abandoned journeys
Document ManagementAPI (via Hub)Customer documents

8. Backbase Implementation

Module: Digital Assist (360 View + Omnichannel Context)

Approach: Configuration (core OOTB capability, integration configuration required)

Key Components: Customer 360 Widget, Activity Timeline Widget, Digital Context Widget, Quick Action Launcher, Notes Widget

Customization Notes: T24 data model mapping, Arabic/English language toggle, DIB-specific quick actions, digital journey event mapping.

9. Definition of Done

  • Customer 360 dashboard loads within 3 seconds for any customer
  • All accounts, products, and balances displayed from T24
  • Digital journey activity visible (last 30 days of app/web sessions)
  • Interaction notes saved and visible across branch, CC, and back office
  • Quick actions launch pre-filled workflows for top 10 common tasks
  • Average inquiry handling time reduced to under 2 minutes

UC-E02

Guided Financing Assist

Priority: P1 Category: Differentiating Module: Digital Assist Capability: Guided Workflows

2. Business Context

Strategic Alignment: Directly enables "Operational Efficiency" and "Personalized Customer Journeys" by streamlining the employee-assisted financing workflow from 45 minutes across 6 systems to 15 minutes in one platform.

Business Value: Reduce financing processing time by 67%, increase branch financing throughput by 3x, reduce training time for financing specialists from 6 weeks to 2 weeks.

Current State: Financing specialists toggle between T24, LOS, credit bureau portal, document management, email, and Excel to process a single application. Manual data entry across systems leads to errors and delays.

Pain Points Addressed:

  • "I enter the same customer data into 4 different systems for one financing application."
  • "I can't tell the customer when their financing will be approved because I can't see the pipeline status."
  • "New hires need 6 weeks to learn the financing process across all systems."

3. User Context

Primary Persona: Rania Mahmoud — Financing Specialist, branch-based, processes 8-12 financing applications per day.

User Goal: Process a customer financing application from intake to approval in a single guided workflow without switching systems.

Success Criteria: Application submitted with all required data and documents, decision received within the appointment, customer leaves with clear next steps.

4. Happy Path Specification

Preconditions: Customer is identified in Digital Assist. Financing product is selected.

Trigger: Employee opens "New Financing" from customer 360 or quick actions.

StepEmployee ActionSystem Response
1Opens customer contextDisplay pre-filled customer data with product recommendation
2Selects product and calculatesReal-time calculator with profit structure and schedule
3Fills application formPre-populated from 360 data, guided step-by-step
4Captures documentsScan/upload with OCR validation and checklist
5Confirms Shariah reviewDisplay compliance check with Fatwa board certification
6Submits for approvalRoute to decision engine, display real-time status

Postconditions: Application submitted, documents stored, decision pending or received, customer notification scheduled.

5. Screen Flow

  • Screen 1: Customer Context — Pre-filled data, product recommendation
  • Screen 2: Product & Calculator — Murabaha/Ijarah selection, real-time calculation
  • Screen 3: Application Form — Guided form with pre-filled fields
  • Screen 4: Document Collection — Checklist with scan/upload
  • Screen 5: Shariah Review — Compliance confirmation
  • Screen 6: Submission & Status — Approval pipeline view

6. Data Requirements

Input Data: Financing amount, tenor, employment details (verified), supporting documents.

System Data: Customer profile (T24), salary history, credit bureau score, product eligibility, Shariah-approved rates.

Output Data: Financing application record, document package, decision request, customer notification.

7. Integration Requirements

SystemIntegration TypePurpose
Temenos T24Real-time API (via Hub)Customer data, salary records, financing creation
Credit Bureau (AECB)Real-time APICredit score and liability check
Decision EngineReal-time APIInstant pre-approval decision
Document ManagementAPI (via Hub)Document storage and retrieval
Notification ServiceEvent-drivenCustomer SMS/push for decision

8. Backbase Implementation

Module: Digital Assist (Guided Workflows)

Approach: Configuration + Extension (Shariah workflow and Murabaha calculation are custom)

Key Components: Guided Workflow Engine, Calculator Widget, Document Capture Widget, Approval Pipeline Widget

Customization Notes: Murabaha/Ijarah product logic, T24 financing creation adapter, AECB integration, Arabic document OCR.

9. Definition of Done

  • Application form pre-populated with 80%+ of customer data from 360 view
  • Profit rate calculator updates in real-time within the guided workflow
  • Document checklist ensures 100% completeness before submission
  • Decision received within 60 seconds for pre-qualified customers
  • End-to-end processing time under 15 minutes (down from 45)
  • New specialist can complete workflow after 2 weeks of training (down from 6)
